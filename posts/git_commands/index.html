Git Commands

Git commands everyone should know which are most commonly used in any software development workflow.

  1. git clone «repo» - This command clones a remote repository onto your local machine.

    E.g.git clone https://github.com/AdityaBagad/Password-Strength-Analysis.git - Password Strength Analysis repository is cloned to your local machine.

  2. git add «files/directory» - This command is used to save most recent changes to staging area so that they can be tracked.

    E.g.git add hello.txt - This command will add hello.txt file to the staging area.

    E.g.git add --all - This command will add files in the current directory. The --allflag in the command specifies that everything under the current directory and the files under that directory should be tracked.

  3. git commit -m «message» - After adding your files to the staging area we have to commit them. We have to add those changes and reflect them in the remote repository.

    E.g.git commit -m "Fixed the Add to Cart UI bug" - The -m flag stands for message. A commit message must be meaningful.

    E.g.git commit --amend - m "My New Commit" - If you accidentally made a mistake in the commit message, you can change it using the --amend flag.

  4. git status - This command allows you to view the files that are being tracked or are in the staging area.

  5. git branch - This command is used to see the list of all branches in your repository or you can create a new branch with this command. Your current active branch is highlighted with a * prefix.

    E.g.git branch - Lists all the branches in your repo.

    E.g.git branch new_feature - Creates a new branch named “new_feature” but it is not switched to the newly created branch.

    E.g.git branch -d new_feature - Delete the “new_feature” branch.

  6. git checkout «branch-name» - This command is used to switch to the branch name specified.

    E.g.git checkout new_feature - Switches to the new_feature branch

    E.g.git checkout -b another_feature - This creates a new branch “another_feature” and it switches to the branch created. The -b flag is used to create a new branch “another_feature”.

  7. git push «remote-name» «branch-name» - This command is used to push your changes into the remote repository from your local repository.

    E.g.git push origin new_feature - This pushes your new_feature branch to remote repository.

  8. git pull - This command is used to get all the latest changes from the remote repository.

    E.g.git pull - This updates your current working local branch i.e. it syncs the remote repository branch to your local machine.
